# 前端页面使用指南

## 概述

前端页面提供了一个完整的可视化测试界面，可以直观地测试后端的所有API功能。

## 访问方式

### 方式1: 直接打开HTML文件

```bash
# Windows
start index.html

# Linux/Mac
open index.html
# 或
xdg-open index.html
```

### 方式2: 使用本地服务器

```bash
# 使用Python
python -m http.server 8000

# 使用Node.js (需要安装http-server)
npx http-server

# 使用PHP
php -S localhost:8000
```

然后访问: http://localhost:8000/index.html

## 页面功能说明

### 📑 标签管理

#### 1. 创建标签
- **标签名称**（必填）：标签的唯一标识
- **标签颜色**：自定义标签显示颜色
- **排序顺序**：控制标签显示顺序
- **标签描述**：标签的详细说明

**使用示例**：
1. 输入标签名称：`重要文档`
2. 选择颜色：`#EF4444`（红色）
3. 设置排序：`1`
4. 点击"创建标签"按钮

#### 2. 标签列表
- 显示所有标签的详细信息
- 支持搜索功能：输入标签名称或描述
- 显示标签使用次数（关联的文件数）
- 每行标签提供查看和删除操作

#### 3. 标签合并
- 将一个标签的所有关联关系合并到另一个标签
- 防止相同标签出现多个实例
- 使用事务保证数据一致性

**使用示例**：
1. 找到要合并的标签A（源标签）
2. 找到目标标签B（要保留的标签）
3. 输入源标签ID和目标标签ID
4. 点击"合并标签"按钮

---

### 📁 文件标签关联

#### 1. 添加单个标签
- 指定文件ID和标签ID
- 为文件添加一个标签

#### 2. 批量添加标签
- 指定文件ID
- 输入多个标签ID（逗号分隔）
- 为一个文件同时添加多个标签

#### 3. 移除单个标签
- 指定文件ID和标签ID
- 从文件移除一个标签

#### 4. 批量移除标签
- 指定文件ID
- 输入多个标签ID（逗号分隔）
- 从文件同时移除多个标签

#### 5. 查询文件标签
- 输入文件ID
- 查看该文件关联的所有标签
- 支持在结果中直接移除标签

**使用示例**：
1. 文件ID：`1`
2. 标签ID：`1`
3. 点击"添加标签"按钮

---

### 🔍 多标签查询

#### 功能说明
- 选择多个标签
- 查询同时具有这些标签的所有文件
- 基于PostgreSQL的JSONB查询

**使用示例**：
1. 选择标签：`高优先级` + `中优先级`
2. 点击"查询同时具有多个标签的文件"
3. 查看查询结果

---

### 📊 统计数据

#### 1. 标签使用统计
- 按使用频率排序显示所有标签
- 显示每个标签关联的文件数量
- 便于了解标签使用情况

#### 2. 标签计数详情
- 批量查询指定标签的使用次数
- 输入多个标签ID（逗号分隔）
- 显示每个标签的详细计数

## API接口映射

| 前端功能 | 后端API | 说明 |
|---------|---------|------|
| 创建标签 | POST /api/labels | 创建新标签 |
| 查询标签列表 | GET /api/labels | 分页查询标签 |
| 更新标签 | PUT /api/labels/{id} | 更新标签信息 |
| 删除标签 | DELETE /api/labels/{id} | 删除标签 |
| 标签合并 | POST /api/labels/merge | 合并标签 |
| 标签统计 | GET /api/labels/statistics | 获取统计信息 |
| 添加标签 | POST /api/file-tags | 添加单个标签 |
| 批量添加 | POST /api/file-tags/batch | 批量添加标签 |
| 移除标签 | DELETE /api/file-tags/{fileId}/{labelId} | 移除单个标签 |
| 批量移除 | DELETE /api/file-tags/{fileId}/batch | 批量移除标签 |
| 查询文件标签 | GET /api/file-tags/{fileId} | 查询文件标签 |
| 多标签查询 | POST /api/file-tags/search/intersection | 交集查询 |
| 标签计数 | GET /api/file-tags/count/{labelId} | 单个标签计数 |
| 批量计数 | GET /api/file-tags/count/batch | 批量标签计数 |

## 使用流程示例

### 完整的标签管理流程

#### 第一步：创建标签
1. 打开"标签管理"标签页
2. 输入标签信息：
   - 标签名称：`紧急任务`
   - 颜色：`#EF4444`
   - 排序：`1`
3. 点击"创建标签"

#### 第二步：添加文件标签
1. 切换到"文件标签关联"标签页
2. 输入：
   - 文件ID：`1`
   - 标签ID：`1`（新创建的标签）
3. 点击"添加标签"

#### 第三步：查询标签统计
1. 切换到"统计数据"标签页
2. 点击"获取统计"
3. 查看标签使用情况

#### 第四步：多标签查询
1. 切换到"多标签查询"标签页
2. 选择标签：`紧急任务` + `待审核`
3. 点击"查询同时具有多个标签的文件"

#### 第五步：标签合并
1. 确认需要合并的标签
2. 输入源标签ID和目标标签ID
3. 点击"合并标签"

## 常见问题

### Q1: 页面无法加载后端数据

**A**: 检查以下几点：
1. 后端服务是否启动（http://localhost:8080/api）
2. API_BASE 常量是否正确
3. 浏览器控制台是否有CORS错误

### Q2: 创建标签后立即消失

**A**: 可能原因：
1. 标签名称重复
2. 数据库未正确连接
3. 服务器返回错误但未显示

**解决方法**：
1. 检查标签名称是否已存在
2. 查看浏览器控制台的错误信息
3. 确认数据库连接正常

### Q3: 标签选择器不显示

**A**:
1. 确保"标签管理"标签页已初始化
2. 刷新页面重试
3. 检查后端API是否返回数据

### Q4: 多标签查询无结果

**A**:
1. 确保已创建足够的测试数据
2. 检查标签ID是否正确
3. 查看是否有数据同时具有选中的标签

## 性能优化建议

1. **批量操作**：优先使用批量API而非多次单次操作
2. **分页查询**：大量数据时使用分页参数
3. **缓存策略**：标签列表等静态数据可缓存
4. **请求去重**：避免重复发送相同请求
