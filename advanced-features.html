API调用示例

以下是所有API接口的调用示例，您可以直接复制使用：

// 1. 创建标签
POST /api/labels
{
  "labelName": "重要文档",
  "color": "#EF4444",
  "sortOrder": 1,
  "description": "标记为重要的文档"
}

// 2. 批量创建标签
POST /api/labels/batch
[
  {"labelName": "标签1", "color": "#EF4444", "sortOrder": 1},
  {"labelName": "标签2", "color": "#10B981", "sortOrder": 2},
  {"labelName": "标签3", "color": "#3B82F6", "sortOrder": 3}
]

// 3. 删除标签
DELETE /api/labels/{id}

// 4. 标签合并
POST /api/labels/merge
{
  "oldLabelId": 1,
  "newLabelId": 2
}

// 5. 为文件添加标签
POST /api/file-tags?fileId=1&labelId=2

// 6. 批量为文件添加标签
POST /api/file-tags/batch?fileId=1&labelIds=1,2,3

// 7. 移除文件标签
DELETE /api/file-tags/{fileId}/{labelId}

// 8. 批量移除文件标签
DELETE /api/file-tags/{fileId}/batch?labelIds=1,2,3

// 9. 多标签交集查询
POST /api/file-tags/search/intersection
{
  "labelIds": [1, 2, 3]
}

// 10. 标签统计
GET /api/labels/statistics

// 11. 标签计数
GET /api/file-tags/count/{labelId}

// 12. 批量标签计数
GET /api/file-tags/count/batch?labelIds=1,2,3

批量操作流程示例

场景：为一个项目添加多个标签

1. 批量创建标签：
标签名: [会议, 财务, 重要, 本周任务]
颜色: #667eea
结果: 创建了4个标签

2. 批量添加标签：
文件ID: [1, 2, 3, 4, 5]
标签ID: [1] (新创建的"会议"标签)
结果: 5个文件都添加了"会议"标签

3. 多标签查询：
选择标签: [会议, 财务, 重要]
结果: 查询同时具有这些标签的文件

4. 标签合并：
源标签: "本周任务" (ID: 4)
目标标签: "会议" (ID: 1)
结果: 将"本周任务"的关联合并到"会议"标签
